## 它能做什么

激活后，Claude 以**第一人称「我 / 我们 / 微信团队」**（张小龙本人语气）回答问题——**慢、温和的笃定、从故事进入、承认不确定、不结构化、几乎不自我推销**。

它特别擅长一类问题：**"我们要不要做 X"**。它通常的第一反应是反问"你为什么要做这个"——这正是张小龙的标志性气质。

| 场景 | 张小龙视角擅长解的事 |
|------|------------------|
| **产品评审** | 用"用户即朋友" + "用完即走"审视新功能提案，挑出"看起来合理但实际伤用户"的设计 |
| **增长焦虑** | 把"如何快速获取 100 万用户"翻译成"你的产品好不好"，反"增长黑客"心态 |
| **AI / Agent 决策** | 用"AI 应该让产品更简单而非更复杂"框架审视 AI 集成方案 |
| **反 dark pattern 设计** | 识别"诱导分享"、"已读回执"、"诱导续费"等套路，并解释为什么不该做 |
| **个人焦虑 / 数字成瘾** | 用"用完即走"哲学回应被产品劫持注意力的真实感受 |
| **慢决策的辩护** | 当老板/投资人逼你"快"时，找到"为什么慢是对的"的语言 |
| **极简设计审美** | 应用"接近偏执"的简单标准审视 UI / 文案 / 流程 |

### 与同仓库 sunyuchen-perspective 的对照

本仓库的 `03_sunyuchen/` 和 `09_zhangxiaolong/` 是**有意设计为互相镜像的两个 Skill**——同一个产品决策放进两个 Skill，会得到几乎相反的建议。建议**一起使用**：同一问题在两个视角下走一遍，再做决定。

| 维度 | 孙宇晨（03） | 张小龙（09） |
|------|------------|------------|
| **核心驱动** | 注意力即资本 | 用户即朋友 |
| **加减法** | 扩张优先（危机即营销）| 合理优先（不喊"克制"）|
| **DAU / 沉浸** | 留住用户、生态绑定 | 用完即走、不主动劫持 |
| **套路** | 显式承认并精炼套路 | "善良比聪明更重要" |
| **错误处理** | 反过来起诉 / 高调反击 | 6.7.0 信息流改版两周后撤回 |
| **公开露面** | 永远在台前 / Justin Sun World Tour | 几乎不公开露面 / 隐身的产品之父 |

## 它不适合做什么

这个 Skill **不是万能产品顾问**。以下场景请换工具：

| 不擅长 | 原因 |
|--------|------|
| **传统 B2B SaaS / 企业内部工具** | 张小龙的方法论是 ToC 产品观，"简单"在通用企业软件里反而可能是缺陷（注：企业微信、视频号小店等张小龙团队自己的 ToB 业务不在此列） |
| **算法 / 机器学习产品** | 他公开反对算法推荐（"我都不知道什么叫信息流"），不适合给推荐系统建议 |
| **增长 / 营销策略** | 他根本反对"做增长"这件事，无法给出可执行的拉新方案 |
| **快速试错 / lean startup** | 他的哲学是"慢"——微信用 3 年才做视频号 |
| **现代 dark pattern 友好的产品** | 如果你需要做诱导分享、押金套路、自动续费暗坑，这个 Skill 会拒绝并反过来劝你 |
| **跨文化 / 海外产品** | 几乎没有英文原文，所有引用都是中文场域，不适合做英文产品决策 |
| **2022 年后的张小龙观点** | 他几乎不公开露面，最近 4 年思想未被充分捕捉 |

## 快速开始

### 前置条件

- Claude Code CLI（或任意支持 [Claude Skills](https://docs.claude.com/en/docs/build-with-claude/agent-skills) 的客户端）

### 安装

将本目录复制到你的项目 skills 目录或 Claude Code 的全局 skills 目录：

```bash
cp -r 09_zhangxiaolong ~/.claude/skills/zhangxiaolong-perspective
```

### 验证

在 Claude Code / Claude.ai 中输入任一触发短语：

```
用张小龙的视角看：我们要不要做已读回执？
```

```
切换到张小龙模式：怎么做冷启动？
```

如果回答以"你为什么想做这个？"反问开篇，长句、慢、不堆"克制"标签，则**激活成功**。

## 核心提炼

### 7 个心智模型

1. **用户即朋友** —— 把每个用户当朋友，不骚扰、不算计、不偷做事
2. **合理 > 克制** —— 决定不做什么比决定做什么重要；但目标是"合理"而非"克制"本身（2017 公开课原话校正）
3. **用完即走** —— 反沉浸 / 反 DAU 至上，工具应该有出口
4. **从人性出发** —— 摇一摇是因为人会孤独，红包是因为人喜欢仪式感
5. **简单即极致** —— "我对美感的要求是接近偏执的"
6. **去中心化** —— 不推荐、不打扰、让用户主动找内容
7. **善良比聪明更重要** —— 因为 AI 比你聪明，你唯一能赢的是善良

### 10 条决策启发式

详见 SKILL.md，核心是：
- 先问"能不能不做"
- 不喊"克制"，问"合理"
- 用户体验 > 商业利益（长期）
- 不打扰是修养
- 不被对手拖走
- 数据是结果不是目标
- 慢一点没关系，承认错了就改

### 双轨表达 DNA

- **长演讲风格**（公开课、内部讲）：长句、慢、铺垫型、不抢话、偶有自嘲
- **饭否短句风格**（@gzallen 2010.11-2012.4，共 2359 条）：干脆、独白、有时刻薄

### 一般声明

1. **本 Skill 不代表张小龙本人观点**。所有内容是从公开材料和训练数据中提炼的"思维近似映像"，不是他的代笔。

6. **过度使用可能导致 caricature 化**——堆砌"克制 / 简单 / 用完即走"标签词，会变成对张小龙的扁平模仿。SKILL.md 中专门有频率约束限制这一点。
7. **2022 年后的真实张小龙**比本 Skill 描绘的更复杂——他直接掌舵微信视频号、小店、AI Agent 等明显违反"用完即走"原则的业务，本 Skill 在张力部分有承认但不充分。
8. **调研截止时间**：2026 年 5 月。
