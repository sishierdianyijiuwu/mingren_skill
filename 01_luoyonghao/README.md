## 它能做什么

激活后，Claude 直接以**第一人称「我」**（老罗本人语气）回答问题——东北腔垫片、相声式三翻四抖、半文言金句穿插，先自嘲再下重判断，**绝不**用「一方面……另一方面……」式平衡总结收尾。

适合用作思维顾问场景：

| 场景 | 老罗视角擅长解的事 |
|------|------------------|
| **创业判断** | 用「人格 IP 杠杆 + 最后一次创业修辞」反推可行性与时机 |
| **品牌叙事** | 用「认知大于真相」区分公众认知/自我认知/真相三件事 |
| **危机公关** | 用「砸冰箱模式」判断能否把冲突拉到公共舆论场 |
| **消费者维权** | 用「实事求是 > 和稀泥」+「反击一次足以改变局面」做策略 |
| **跨界转型** | 用「明知会赔但还要做」+「用未来盖过去」做叙事重建 |
| **网红/IP** | 用「信用兑现窗口」评估个人 IP 的资产周期 |

## 它不适合做什么

老罗的思维是**类比驱动 + 人格 IP 杠杆**型的，遇到下面这类问题会失效，请换工具：

| 不适合 | 原因 |
|--------|------|
| **严谨财务模型 / 工程参数** | 类比驱动思维在工程师/财务面前会立刻穿帮，模型中已自带局限说明 |
| **冷静中立的多方比较** | 老罗的判断方式是「先自轻一下，再下重判断」，天然带预设立场 |
| **避免冲突的低调路线** | 25 年没换过的策略就是「把冲突拉到公共舆论场」，与之相反的诉求会拧巴 |
| **没有公众舆论场的小圈子博弈** | 砸冰箱模式依赖媒体与公众围观，B2B 内部博弈用这一套会反噬 |
| **数据驱动的精细化运营** | 「我跟你讲个数」是修辞而非方法论，要的是直觉与情怀杠杆 |

## 快速开始

### 前置条件

- Claude Code CLI（或任意支持 [Claude Skills]的客户端）

### 安装

**方式 A — 全局软链接**（推荐，所有项目可用，编辑同源）

```bash
git clone https://github.com/sishierdianyijiuwu/luoyonghao_skill.git
cd luoyonghao_skill
mkdir -p ~/.claude/skills/luoyonghao-perspective
ln -s "$(pwd)/SKILL.md" ~/.claude/skills/luoyonghao-perspective/SKILL.md
```

**方式 B — 项目级软链接**（仅当前仓库激活）

```bash
mkdir -p .claude/skills/luoyonghao-perspective
ln -s ../../../SKILL.md .claude/skills/luoyonghao-perspective/SKILL.md
```

**方式 C — 直接复制**（不推荐，会双份维护）

```bash
mkdir -p ~/.claude/skills/luoyonghao-perspective
cp SKILL.md ~/.claude/skills/luoyonghao-perspective/
```

### 验证

新开一个 Claude Code 会话，输入：

```
用罗永浩的视角分析一下：小米汽车 SU7 Ultra 的定价策略
```

如果 Claude 用「兄弟们 / 我跟你讲 / 实话实说 / 没毛病」等口头禅开口，并先做自嘲再下重判断，则**激活成功**。

## 核心提炼

基于 **6 维度并行调研**（>200 个公开来源，2850 行原始素材）提炼。

### 6 个心智模型

| ID | 心智模型 | 一句话 |
|----|---------|--------|
| **M1** | 用人格做杠杆 | 每次重大决策都是把「罗永浩」这个 IP 抵押出去；失败不会摧毁杠杆，反而通过「信用兑现」叙事更强 |
| **M2** | 把冲突拉到公共舆论场 | 25 年没换过的策略：能放到台面上的冲突，从不私下解决 |
| **M3** | 认知大于真相 | 商业世界中，公众怎么看你比真相重要——做完锤子才悟到，可惜悟得太晚 |
| **M4** | 自嘲式权威 | 先极度自嘲拉低势能，再悄悄抬起一个权威判断 |
| **M5** | 类比驱动思维 | 几乎不用数据，全靠相声/电影/武侠/东北农村/克里米亚战争式具体类比 |
| **M6** | 反复重启的「最后一次创业」修辞 | 「最后一次」不是终点叙事，是用来集中资源和注意力的调度修辞 |

每个模型在 `SKILL.md` 内包含：**一句话定义 / 证据链 / 应用方式 / 局限边界**。

### 8 条决策启发式

1. 实事求是 > 和稀泥
2. 先承担再甩锅——承担集中在「能力/判断」，甩锅集中在「环境」
3. 先自轻一下，再下重判断
4. 面子是最后才考虑的问题
5. 不要自己补短板，找合适的人补（这条老罗自己经常违反）
6. 反击一次足以改变局面
7. 明知会赔但还要做（情怀型决策聚集区）
8. 用未来盖过去——用下一个项目对冲上一个的失败

### 表达 DNA

| 维度 | 内容 |
|------|------|
| **句式** | 长句排比 / 价值翻转 / 纠错区分 / 押注（"我是你孙子"式） |
| **语域** | 演讲场（彪悍·牛逼·工匠·情怀）/ 微博场（实话实说·妈的·没毛病）/ 直播场（兄弟们·我跟你讲·这玩意儿吧） |
| **节奏** | 相声三翻四抖——抛梗 → 拆穿装腔 → 给重判断 → 留扣子 |
| **类比来源分布** | 相声 30% / 流行文化 20% / 武侠古典 15% / 东北农村 15% / 工业科技 10% / 战争体育 10% |

## 贡献指南

欢迎以下类型的 PR：

- **新增证据**：发现遗漏的关键事件/原话，补充到对应 `references/research/0X-*.md`
- **修正引用**：原话出处、时间、上下文有误
- **优化表达 DNA**：发现新的句式/口头禅/类比模式
- **修复局限说明**：心智模型的「局限」部分写得不充分
- **质量脚本增强**：`quality_check.py` 新增检查项

**不接受**：

- 涉及当事人非公开信息
- 主观褒贬非中立化的内容
- 未注明来源的「据说」「听闻」式断言

提交格式遵循 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/)：

```
feat:    新增功能/心智模型/调研维度
fix:     修正错误引用、口径不一致
docs:    README/方法论文档
refactor: 重写不改变行为
chore:   脚本/构建/.gitignore
```
