## 它是什么 / 不是什么

### 它是什么

一组**可激活的"思维操作系统"**。每个 Skill 让 Claude 用一个真实人物的**心智模型 + 决策启发式 + 表达 DNA** 来回应你的问题——不是模仿语气，是提炼框架。

你可以把它当作：
- **思维顾问**：用罗永浩 / 雷军 / 孙宇晨…的镜片审视自己的决策
- **多视角辩论**：同一个问题让两个不同 Skill 各自给一个判断
- **写作辅助**：模拟某个人物的发言、文章、内部信
- **决策对照**：评估自己的选择能不能"通过"某个企业家心里那一关

### 它不是什么

- **不是粉丝向角色卡**——不会"装雷军安慰你""装罗永浩骂世界"
- **不是真人代理**——任何 Skill 都自带免责声明，不代表当事人本人观点
- **不是 PR 复读机**——每个 Skill 都明确写出当事人的内在张力、争议、信息盲区
- **不是娱乐玩具**——这是一套思维提炼工具，娱乐价值是副产品

## 快速开始

### 1. 选一个 Skill 安装

以雷军 skill 为例：

```bash
git clone https://github.com/sishierdianyijiuwu/mingren_skill.git
cd mingren_skill/02_leijun

# 全局软链接（推荐）
mkdir -p ~/.claude/skills/leijun-perspective
ln -s "$(pwd)/SKILL.md" ~/.claude/skills/leijun-perspective/SKILL.md
```

罗永浩 skill 同理，目录改成 `01_luoyonghao` 即可。

### 2. 在 Claude Code 里激活

```
用雷军的视角分析一下：35 岁该不该离职做硬件创业？
```

或

```
让老罗骂醒我，我这个 SaaS 想法靠不靠谱？
```

激活后 Claude 会用对应人物的第一人称、心智模型、表达 DNA 回应。

### 3. 同时激活多个 Skill（思维对照）

```
我想做一款面向中国 Z 世代的硬件产品。
请分别用雷军的视角和罗永浩的视角各给一段判断，再总结二者的核心分歧。
```

Claude 会切换两副镜片各回一段，最后做差异综合。这是这套 Skill 工坊**最有价值的用法**——同一现实，两种思维操作系统并行运行，分歧本身就是洞察。

## 设计理念

蒸馏一个人物的思维框架，有 4 条品味原则我们一直在守：

### 1. 长文 > 金句

3000 字 essay 比 50 条推文更能揭示思维结构。年度演讲、亲笔著作、深度访谈是核心语料——微博 / 抖音只是表达 DNA 的补充。

### 2. 争议 > 共识

最被争议的观点最能揭示独特性。每份 Skill 都包含「他者视角」维度（Agent 4），收集主流叙事不讲的批评、对手观察、离职高管侧写。

### 3. 变化 > 固定

改变立场的地方比一直坚持的更有信息量。**立场转变轨迹**是蒸馏的核心交付物之一：罗永浩从锤子到交个朋友的杠杆迁移、雷军从「不造车」到押 100 亿造车的 7 年反转，都被完整记录。

### 4. 张力 > 自洽

每个 Skill 都明确写出至少 3-6 对**内在张力**（如雷军的「梦想 vs 求生」「工程师文化 vs 营销大师」），不为人物 PR 抹平矛盾。**矛盾是人格的核心特征，不是需要修复的 Bug**。

## 贡献指南

欢迎下列类型的 PR：

### 对已有 Skill 的改进
- **新增证据**：发现遗漏的关键事件 / 原话 → 补到对应 `references/research/0X-*.md`
- **修正引用**：原话出处 / 时间 / 上下文有误（一手来源订正"语录"洗稿是最高优先级）
- **更新最新动态**：人物最新立场转变 / 重大决策（雷军年度演讲 / 老罗新动作）
- **优化表达 DNA**：发现新句式 / 自嘲模板 / 高频词
- **修复局限说明**：心智模型的「局限」部分写得不充分

### 不接受的内容

- 涉及当事人**非公开信息**（家庭 / 个人财富细节 / 内部权力博弈匿名爆料）
- 主观褒贬非中立化内容（特别是粉黑两极的内容）
- 来自**信息源黑名单**的引用：
  - 知乎（洗稿严重）
  - 非官方微信公众号
  - 百度百科 / 百度知道
  - 各类「XX 语录大全」自媒体（基本是杜撰拼凑）

提交格式遵循 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/)：

```
feat:   新增人物 skill / 心智模型 / 调研维度
fix:    修正错误引用、口径不一致
docs:   README / 方法论文档
chore:  脚本、构建、.gitignore
```

## 免责声明

1. 本仓库所有 Skill 均基于人物**公开言论**（演讲、著作、访谈、社交媒体、播客、直播）反推其思维框架。
2. **不代表任何当事人本人观点**，每个 Skill 激活时都会主动声明一次。
3. 所有引用均可溯源至对应 skill 的 `references/research/` 目录。
4. 仅供**思维方式的角色扮演、议题讨论与写作辅助**使用，**不构成商业建议、法律建议、投资建议或行为建议**。
5. 引用的第三方人物、品牌、事件均为公开信息。如认为存在不当引用或侵权，请提 Issue，**会在 7 天内核查处理**。
6. 本仓库与所收录人物本人及其关联商业实体（小米集团 / 顺为资本 / 交个朋友 / 细红线科技 / TRON DAO 等）**均无任何关联**，纯属第三方研究 / 角色扮演项目。
7. **人物的真实信念与公开叙事之间存在差距**——这是中文企业家普遍的特征。Skill 已尽力捕捉这种张力（参见每个 skill 的「核心张力」段），但仍可能存在系统性偏差，使用时建议交叉参考多个视角。
